# 2026-04-08

### 11:30 - Added Teams image support to hive-m365 MCP

- **Problem**: Teams messages with inline images (pasted screenshots, photos) were invisible — `get_chat_messages` returned raw JSON without image content, `get_channel_messages` stripped all HTML including image references
- **Solution**: Added two-layer image support to hive-m365 MCP server:
  - `graph.js`: Added `graphFetchBinary()`, `parseMessageAttachments()`, `getChatMessageHostedContent()`, `getChannelMessageHostedContent()`
  - `index.js`: Modified `get_chat_messages`, `get_channel_messages`, `get_channel_message_replies` to auto-download images for the 5 most recent messages (returned as MCP ImageContent base64 blocks)
  - Added `get_message_image` tool for on-demand fetching of older message images
- **Design decision**: Auto-fetch latest 5 messages' images inline; older ones return metadata only (hostedContentId) to avoid slow responses when many images exist
- **Files changed**: `Hive/mcp/hive-m365/src/graph.js`, `Hive/mcp/hive-m365/src/index.js`
- **Status**: Code done, syntax verified. Needs `mc` restart to take effect
